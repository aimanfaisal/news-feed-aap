🏭 Factory Pattern

Creates objects without exposing instantiation logic to the client.

In this app: NotificationFactory.create('Email', ...) returns an EmailNotification object. To add a new type, only edit the factory.

class NotificationFactory:     _creators = {         'Email': EmailNotification,         'SMS':   SMSNotification,         'Push':  PushNotification,     }     @staticmethod     def create(type, message, category):         return _creators[type](message, category)

👁 Observer Pattern

Defines a one-to-many dependency so that when one object changes state, all its dependents are notified automatically.

In this app: NewsPublisher notifies all subscribed UserSubscribers when news is published. Users can subscribe/unsubscribe freely.

class NewsPublisher:   # Subject     def subscribe(self, observer): ...     def publish(self, notification):         for obs in self._subscribers:             obs.update(notification)   class UserSubscriber(Observer):  # Observer     def update(self, notification):         self.received.append(notification)
